# 또박또박 (ddobakddobak)

### AI 회의록 자동 생성 서비스

> "회의에 집중하세요. 기록은 또박또박이 합니다."

---

## 1. 프로젝트 소개

### 해결하려는 문제

- 회의 중 필기에 집중하다 보면 **논의에 참여하기 어려움**
- 회의 후 회의록 정리에 **30분~1시간 추가 소요**
- 기존 클라우드 회의록 서비스는 **데이터 외부 유출 우려**
- 구독형 서비스는 **지속적 비용 발생**

### 또박또박의 해결책

| 문제 | 해결 |
|------|------|
| 필기로 인한 회의 참여 저하 | 실시간 STT가 자동으로 기록 |
| 회의록 정리 시간 소모 | AI가 요약·결정사항·Action Item 자동 생성 |
| 데이터 외부 유출 우려 | 로컬/사내 서버에서 모든 데이터 처리·저장 |
| 구독 비용 | 오픈소스 AI 모델 활용, 일회성 구축 |

### 대상 사용자

- 정기적으로 대면 회의를 진행하는 **소규모 팀 (2~10명)**
- **데이터 보안**이 중요하여 로컬 저장을 선호하는 조직
- 회의록 작성에 시간을 많이 소비하는 팀

## 2. 핵심 기능

### 2.1 실시간 음성 인식 (STT)

- 브라우저 마이크로 녹음하면 **즉시 텍스트로 변환**
- Voice Activity Detection (VAD)으로 **무음 구간 자동 건너뛰기**
- AudioWorklet 기반 **저지연 오디오 처리**
- 설정 가능한 청크 크기, 오버랩 지원
- **9개 언어 지원**: 한국어, English, 日本語, 中文, Español, Français, Deutsch, ไทย, Tiếng Việt

**STT 엔진 (Adapter 패턴으로 교체 가능):**

| 엔진 | 특징 | 적합 환경 |
|------|------|-----------|
| Qwen3-ASR 1.7B | CJK 최고 정확도, MLX Metal 가속 | macOS Apple Silicon |
| Whisper Large v3 Turbo | 범용, C++ 최적화 | 모든 플랫폼 |
| Faster Whisper | CUDA GPU 병렬 처리 | NVIDIA GPU 서버 |

> 환경 변수 하나(`STT_ENGINE`)로 엔진 전환 가능

### 2.2 AI 화자 분리

- **pyannote.audio** 기반 발화자 자동 구분
- 음성 임베딩 DB로 **동일 인물 자동 인식** (회의 내/회의 간)
- 화자 이름 변경 및 관리 기능
- 설정에서 on/off 전환 가능

### 2.3 AI 회의록 자동 생성

- **실시간 중간 요약** (15초~5분 간격, 설정 가능)
- 회의 종료 시 **최종 요약 자동 생성**
- 구조화된 결과물:
  - 핵심 요약
  - 주제별 논의 사항
  - 결정 사항
  - Action Item (담당자 + 기한)
- **AI 피드백**: "결정사항을 표로 정리해줘" 등 자연어로 수정 요청 가능

**LLM 프로바이더 (선택 가능):**
- Anthropic API (Claude)
- OpenAI API 호환
- 사내 LLM (Z.ai GLM)
- Ollama (로컬 LLM)

### 2.4 블록 에디터

- **Notion 스타일** WYSIWYG 편집 (BlockNote 기반)
- AI 요약 결과를 직접 편집 가능
- **Mermaid 다이어그램** 자동 렌더링
- 텍스트, 제목, 테이블, 이미지, 코드 블록 지원

### 2.5 오디오 파일 업로드

- 녹음 파일 (mp3, wav, m4a, flac, ogg) 업로드로 회의록 생성
- 대용량 파일 분할 처리
- 업로드 진행률 실시간 표시

### 2.6 기타 기능

- **대시보드**: 회의 현황 통계, 최근 회의 목록
- **폴더 관리**: 계층적 폴더 구조, 드래그앤드롭 이동
- **태그**: 회의 분류 및 검색
- **첨부파일**: 회의에 관련 파일 첨부
- **내보내기**: Markdown, Word(docx) 형식 지원
- **오디오 플레이어**: 파형 시각화, 전사 텍스트와 시간 동기화
- **팀 협업**: 팀 생성, 멤버 초대, 회의록 공유
- **프롬프트 템플릿**: 회의 유형별 AI 요약 프롬프트 커스터마이징

## 7. 기술적 특징

### 7.1 STT Adapter 패턴

```python
class SttAdapter:                    # 공통 인터페이스
    def transcribe(audio) -> Result
    def get_info() -> EngineInfo

class Qwen3Adapter(SttAdapter): ...  # MLX Metal 가속
class WhisperAdapter(SttAdapter): ... # C++ 최적화
class FasterWhisperAdapter(SttAdapter): ... # CUDA GPU
```

- 환경 변수 하나로 엔진 교체: `STT_ENGINE=qwen3_asr_8bit`
- 새 모델 추가 시 Adapter 클래스 하나만 구현

### 7.2 VAD (Voice Activity Detection) 기반 청킹

![VAD 청킹 흐름](diagrams/05-vad-chunking.svg)

<details>
<summary>텍스트 버전 보기</summary>

```
연속 음성 스트림
    ↓ AudioWorklet (브라우저)
무음 감지 → 청크 경계 결정
    • 무음 0.5초 이상 → 청크 분리
    • 최소 2초 ~ 최대 10초 청크
    • 0.5초 오버랩으로 경계 음절 손실 방지
    ↓
최적 크기 오디오 청크 → STT 전송
```

</details>

## 10. 실행 방법

```bash
# 1. Rails API 서버
cd backend && bundle install && rails db:setup
bundle exec rails s -p 13323

# 2. Python Sidecar (AI 엔진)
cd sidecar && uv sync --extra macos
uv run uvicorn app.main:app --port 13324

# 3. 프론트엔드
cd frontend && npm install && npm run dev

# 4. (선택) Tauri 데스크톱 앱
cd frontend && npm run tauri:dev
```
