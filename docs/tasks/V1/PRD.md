# PRD: 또박또박 (ddobakddobak)

> 회의 음성을 실시간으로 텍스트화하고, 종료 후 핵심 요약·결정사항·할 일을 자동으로 정리해 공유하는 회의 보조 앱

**문서 버전:** v1.0
**작성일:** 2026-03-24
**상태:** Draft

## 1. 개요

### 1.1 프로젝트 비전

대면 회의에서 발생하는 음성을 실시간으로 텍스트로 변환하고, AI가 자동으로 핵심 요약·결정사항·Action Item을 정리하여 팀원과 공유할 수 있는 로컬 기반 회의 보조 웹앱.

### 1.2 핵심 가치

- **실시간성**: 회의 중 음성이 즉시 텍스트로 변환되어 화면에 표시
- **자동 정리**: AI가 회의 내용을 구조화하여 요약·결정사항·할 일을 자동 생성
- **프라이버시**: 모든 데이터(음성, 텍스트, 요약)를 로컬에 저장, 외부 유출 없음
- **협업**: 팀원과 회의록을 앱 내에서 공유하고 함께 편집

### 1.3 대상 사용자

- **소규모 팀 (2~10명)**
- 정기적으로 대면 회의를 진행하는 팀
- 회의록 작성에 시간을 많이 소비하는 팀
- 데이터 보안이 중요하여 로컬 저장을 선호하는 팀

### 2.1 STT 모델 추상화 (Adapter 패턴)

STT 엔진은 **플러그인 방식**으로 설계하여, 설정 변경만으로 모델을 교체할 수 있도록 한다.

**설계 원칙:**
- 모든 STT 엔진은 공통 인터페이스(`SttAdapter`)를 구현
- 환경 변수(`STT_ENGINE`)로 사용할 엔진을 선택
- 새 모델 추가 시 Adapter 하나만 구현하면 됨

```
# 공통 인터페이스 (Python Sidecar)
class SttAdapter:
    def transcribe(audio_chunk) -> TranscriptSegment
    def start_stream() -> StreamHandle
    def stop_stream(handle) -> FinalTranscript

# 환경 변수로 엔진 선택
STT_ENGINE=qwen3_asr       # 기본값
STT_ENGINE=whisper_cpp      # 대안 1
STT_ENGINE=faster_whisper   # 대안 2
STT_ENGINE=sensevoice       # 대안 3
```

**지원 예정 엔진:**

| 엔진 | 한국어 정확도 | Apple Silicon 속도 | 비고 |
|------|-------------|-------------------|------|
| **Qwen3-ASR-1.7B** (기본) | 최우수 (SOTA) | 빠름 (vLLM/mlx-lm) | 2026.01 공개, Apache 2.0 |
| whisper.cpp (large-v3-turbo) | 매우 좋음 | 최고 (Metal/ANE) | Apple Silicon 최적화 우수 |
| Faster Whisper | 매우 좋음 | 보통 (CPU only) | pyannote 통합 용이 |
| SenseVoice (Alibaba) | 우수 (CJK 특화) | 빠름 (경량) | 234M 파라미터 |

**OS별 추론 라이브러리:**

STT 및 LLM 추론에 사용되는 라이브러리는 OS/하드웨어에 따라 다르다.

| 라이브러리 | macOS (Apple Silicon) | Linux (NVIDIA GPU) | 용도 |
|-----------|----------------------|-------------------|------|
| **mlx-lm** | ✅ 사용 | ❌ 미지원 | Qwen3-ASR, LLM 추론 (Metal 가속) |
| **vLLM** | ❌ 미지원 (CUDA 필요) | ✅ 사용 | Qwen3-ASR, LLM 추론 (CUDA 가속) |
| **pywhispercpp** | ✅ 사용 (Metal/ANE) | ✅ 사용 (CUDA) | Whisper STT |
| **pyannote.audio** | ✅ 사용 | ✅ 사용 | 화자 분리 |

- macOS 개발 환경: `mlx-lm` + `pywhispercpp`
- Linux 프로덕션 환경: `vLLM` + `pywhispercpp`
- Qwen3 Adapter는 내부에서 OS를 감지하여 적절한 추론 백엔드를 자동 선택한다

**Qwen3-ASR-1.7B를 기본 엔진으로 선정한 근거:**
- 오픈소스 ASR 중 SOTA 성능 달성
- 언어 감지 정확도: **97.9%** (Whisper 94.1%)
- 중국어 WER: **4.97** (Whisper 15.30) — CJK 계열 언어에서 큰 격차
- 영어 Tedlium WER: **4.50** (Whisper 7.69)
- 52개 언어/방언 지원 (한국어 포함)
- Apache 2.0 라이선스 (무료, 상업 사용 가능)

## 3. MVP 기능 명세 (Phase 1)

### 3.1 실시간 음성→텍스트 변환

**우선순위: P0 (필수)**

| 항목 | 상세 |
|------|------|
| 설명 | 마이크 입력 음성을 실시간으로 텍스트로 변환하여 화면에 표시 |
| 입력 | 브라우저 마이크 (Web Audio API → WebSocket으로 서버 전송) |
| STT | STT Adapter 패턴 (기본: Qwen3-ASR-1.7B, 환경 변수로 교체 가능) |
| 언어 | 한국어 전용 |
| 화자 분리 | pyannote.audio로 발언자 구분, "화자 1", "화자 2" 등으로 표시 |
| 출력 | 실시간 텍스트 스트림 + 타임스탬프 |

**사용자 흐름:**
1. "회의 시작" 버튼 클릭
2. 브라우저 마이크 권한 허용
3. 음성이 실시간으로 텍스트로 변환되어 화면에 표시
4. 화자별로 다른 색상/라벨로 구분
5. "회의 종료" 버튼으로 녹음 중지

### 3.2 AI 요약 및 정리

**우선순위: P0 (필수)**

| 항목 | 상세 |
|------|------|
| 설명 | AI가 회의 내용을 분석하여 핵심 요약, 결정사항, Action Item을 자동 생성 |
| 모델 | ZAI GLM 또는 Ollama 로컬 LLM |
| 실시간 요약 | 회의 중 5분 간격으로 중간 요약 업데이트 |
| 최종 요약 | 회의 종료 시 전체 내용 기반 최종 요약 생성 |

**AI 요약 출력 구조:**
```
## 핵심 요약
- 회의의 주요 논의 사항 3~5개 불릿

## 결정사항
- 회의에서 확정된 사항 목록

## Action Items
- [ ] 할 일 내용 (@담당자, 마감: YYYY-MM-DD)
- [ ] 할 일 내용 (@담당자, 마감: YYYY-MM-DD)

## 주요 논의 내용
- 논의 주제별 상세 내용
```

**Action Item 세부:**
- 담당자 지정 (팀원 목록에서 선택)
- 마감일 설정
- 완료/미완료 체크박스
- AI가 자동 추출 → 사용자가 수정 가능

### 3.3 블록 기반 에디터

**우선순위: P0 (필수)**

| 항목 | 상세 |
|------|------|

| 자동 구성 | STT 텍스트 → 화자별 블록으로 자동 구성 |
| AI 요약 | AI 요약 결과가 블록 형태로 자동 삽입 |

### 3.4 회의 관리

**우선순위: P0 (필수)**

- 회의 생성 (제목, 참석자, 날짜/시간)
- 회의 목록 (최신순, 검색)
- 회의 상세 보기 (텍스트 전문, AI 요약, Action Items)
- 녹음 원본 오디오 파일 저장 및 재생
- 오디오 타임라인과 텍스트 동기화 (클릭 시 해당 시점 재생)

### 3.5 사용자/팀 관리

**우선순위: P0 (필수)**

- 이메일/비밀번호 회원가입 및 로그인
- 팀 생성/초대/관리
- 팀원 목록 (Action Item 할당용)
- 회의록 앱 내 공유 (팀원에게 링크 공유)

### 3.6 Markdown 내보내기

**우선순위: P1 (중요)**

- 회의록 전체를 Markdown 파일로 내보내기
- AI 요약 포함/제외 선택
- 원본 텍스트 포함/제외 선택

## 4. Phase 2 기능 (2차 개발)

**우선순위: P2**

- Zoom, Google Meet, Teams 등 시스템 오디오 캡처

### 4.3 녹음 파일 업로드

**우선순위: P2**

- 사전 녹음된 오디오 파일(mp3, wav, m4a) 업로드
- 업로드 후 STT + AI 요약 일괄 처리

### 4.4 실시간 협업 편집

**우선순위: P2**

- 여러 팀원이 동시에 같은 회의록을 편집

**우선순위: P3**

- Google Calendar / Outlook 연동
- 캘린더 회의 일정 자동 가져오기
- 회의 시작 시 자동으로 해당 일정과 매핑

### 4.6 Slack 연동

**우선순위: P3**

- 회의 종료 후 Slack 채널에 AI 요약 자동 전송
- Slack에서 회의록 링크 바로 열기

### 4.7 PDF 내보내기

**우선순위: P3**

- 회의록을 포맷팅된 PDF 문서로 내보내기
- 회사 로고/헤더 커스터마이징

### 4.8 한/영 혼용 지원

**우선순위: P3**

- 한국어 + 영어가 섞인 회의 지원
- 언어 자동 감지 및 전환

### 4.9 회의 템플릿

**우선순위: P3**

- 주간 회의, 스프린트 리뷰, 1:1 등 템플릿 제공
- 커스텀 템플릿 생성/저장

### 4.10 Action Item 대시보드

**우선순위: P3**

- 전체 회의에서 나온 Action Item 통합 대시보드
- 담당자별, 상태별 필터링
- 마감일 임박 알림

### 4.11 회의 통계

**우선순위: P3**

- 화자별 발언 시간/비율 통계
- 월간 회의 횟수, 총 시간 리포트
- 반복되는 주제 트렌드 분석

- Firefox 90+
- Web Audio API, WebSocket 필수

## 8. 리스크 및 대응

| 리스크 | 영향 | 대응 방안 |
|--------|------|----------|
| STT 모델 실시간 지연 | STT 텍스트 표시 지연 | Adapter 패턴으로 모델 즉시 전환 가능 (환경 변수 변경), 청크 크기 조절 |
| 화자 분리 정확도 낮음 | 화자 라벨 오매칭 | 사전 화자 등록(voice enrollment) 기능 추가 검토 |
| 로컬 LLM 요약 품질 | 요약 정확도 부족 | 프롬프트 엔지니어링 최적화, 모델 교체 용이하게 추상화 |

| SQLite 동시 쓰기 | 다수 접속 시 lock | WAL 모드 활성화, 쓰기 큐잉 |

## 9. 용어 정의

| 용어 | 설명 |
|------|------|
| STT (Speech-to-Text) | 음성을 텍스트로 변환하는 기술 |
| 화자 분리 (Speaker Diarization) | 음성에서 각 발언자를 구분하는 기술 |

| CER (Character Error Rate) | 문자 오류율, STT 정확도 지표 |
