# PRD: 또박또박 v3 — 모바일 반응형 웹

> 기존 데스크톱 전용 UI를 모바일/태블릿에서도 사용할 수 있도록 반응형 웹으로 전환한다.

**문서 버전:** v3.0
**작성일:** 2026-04-04
**상태:** Draft
**이전 버전:** [v2.0 PRD](../v2/PRD.md)

---

## 1. 개요

### 1.1 배경

V2에서 서버 배포로 전환되면서, 클라이언트는 더 이상 Tauri 데스크톱 앱에 종속되지 않는다.
서버 URL로 접속하면 브라우저에서도 동작하지만, 현재 프론트엔드는 데스크톱 전용으로 설계되어 있어 모바일 환경에서 사용이 불가능하다:

- 사이드바가 항상 240px를 차지하여 모바일에서 콘텐츠 영역 부족
- 회의 상세/라이브 페이지가 3컬럼 수평 패널(`react-resizable-panels`)로 구성 — 모바일에서 사용 불가
- 터치 타겟이 데스크톱 기준 (24~28px)으로 모바일 조작 불편
- `h-screen` + `overflow-hidden` 패턴이 모바일 브라우저 주소줄과 충돌
- 반응형 breakpoint 사용이 2개 페이지(DashboardPage, MeetingsPage)의 그리드에만 적용

### 1.2 V3 목표

| 목표 | 설명 |
|------|------|
| **모바일 사용 가능** | 스마트폰(375px~)에서 핵심 기능(회의 목록, 회의 상세, 실시간 전사 보기) 사용 가능 |
| **태블릿 최적화** | 태블릿(768px~)에서 2컬럼 레이아웃 등 중간 단계 레이아웃 제공 |
| **데스크톱 유지** | 기존 데스크톱(1024px~) 사용 경험은 그대로 유지 |
| **점진적 전환** | 기존 코드를 최대한 재활용하며, 반응형 클래스 추가 방식으로 전환 |

### 1.3 대상 사용자 및 시나리오

| 시나리오 | 디바이스 | 설명 |
|---------|---------|------|
| **회의 중 실시간 보기** | 스마트폰 | 회의 참석자가 자기 폰으로 실시간 전사 확인 |
| **이동 중 회의록 확인** | 스마트폰 | 과거 회의 요약/전사 조회 |
| **태블릿 회의 운영** | 태블릿 | 태블릿으로 녹음 + 전사 확인 + 메모 |
| **데스크톱 풀 작업** | 데스크톱/노트북 | 기존과 동일한 3컬럼 레이아웃 |

### 1.4 범위

**포함:**
- 프론트엔드 반응형 레이아웃 전환 (CSS/컴포넌트)
- 모바일 내비게이션 패턴 (바텀 내비/햄버거 메뉴)
- 터치 최적화 (터치 타겟, 제스처)
- 모바일 브라우저 대응 (viewport, safe area)

**제외:**
- PWA (서비스워커, manifest) — V4에서 별도 검토
- 네이티브 앱 (Tauri Mobile / React Native) — 별도 프로젝트
- 백엔드 API 변경 — 없음 (기존 API 그대로 사용)
- 모바일 마이크 녹음 — 별도 WP로 검토 (Web Audio API 모바일 호환성 이슈)

## 2. 디자인 원칙

### 2.1 반응형 전략: Desktop-First 유지 + Mobile 추가

기존 데스크톱 UI를 깨뜨리지 않으면서, Tailwind breakpoint를 활용해 모바일 레이아웃을 추가한다.

```
Mobile (< 640px)    : 1컬럼, 바텀 내비, 탭 전환
Tablet (640~1023px) : 2컬럼, 사이드바 오버레이
Desktop (≥ 1024px)  : 기존 레이아웃 유지 (3컬럼 패널, 사이드바)
```

### 2.2 Tailwind Breakpoint 매핑

| Breakpoint | 너비 | 디바이스 | 비고 |
|-----------|------|---------|------|
| (기본) | 0~639px | 스마트폰 | 모바일 우선 스타일 기준 |
| `sm` | 640px~ | 대형 스마트폰/소형 태블릿 | 2컬럼 그리드 시작 |
| `md` | 768px~ | 태블릿 | 사이드바 오버레이 가능 |
| `lg` | 1024px~ | 데스크톱 | 기존 3컬럼 패널 활성화 |
| `xl` | 1280px~ | 대형 데스크톱 | 4컬럼 그리드 |

## 3. 기능 명세

### 3.1 반응형 내비게이션 (P0)

현재 사이드바(`w-60`)가 항상 표시되어 모바일에서 콘텐츠 영역이 부족하다.

#### 3.1.1 브레이크포인트별 동작

| 브레이크포인트 | 내비게이션 | 동작 |
|-------------|----------|------|
| 모바일 (< `lg`) | 바텀 내비게이션 바 | 홈, 회의, 검색, 설정 4개 아이콘 |
| 모바일 (< `lg`) | 사이드바 오버레이 | 폴더/태그 접근 시 오버레이 슬라이드 + 백드롭 |
| 데스크톱 (≥ `lg`) | 기존 사이드바 | 현재와 동일 (`w-60` 고정) |

#### 3.1.2 바텀 내비게이션 구성

```
┌─────────────────────────────────────┐
│          (페이지 콘텐츠)              │
├─────────────────────────────────────┤
│  🏠 홈    📋 회의    🔍 검색    ⚙ 설정  │
│  Home    Meetings  Search  Settings │
└─────────────────────────────────────┘
```

- 높이: 56px (안전 영역 패딩 포함 시 최대 ~80px)
- 아이콘: lucide-react (기존 아이콘셋 활용)
- 활성 상태: 색상 변경 + 라벨 표시

#### 3.1.3 요구사항

| 항목 | 설명 |
|------|------|
| 바텀 내비 | `lg:` 미만에서만 표시, `lg:` 이상에서 숨김 |
| 사이드바 | `lg:` 이상에서만 고정, 모바일에서는 오버레이(sheet) |
| 오버레이 | 폴더 트리, 태그 필터 접근 시 좌측에서 슬라이드 |
| 백드롭 | 오버레이 외부 클릭/터치로 닫기 |
| Safe Area | `pb-safe` (iOS 하단 홈 인디케이터 영역) |

### 3.2 회의 상세 페이지 모바일 레이아웃 (P0)

현재 3컬럼 수평 패널(전사 25% | AI 요약 45% | 메모 30%)이 모바일에서 사용 불가.

#### 3.2.1 브레이크포인트별 레이아웃

| 브레이크포인트 | 레이아웃 |
|-------------|---------|
| 모바일 (< `lg`) | **탭 전환**: 전사 / AI 요약 / 메모 3개 탭 |
| 데스크톱 (≥ `lg`) | 기존 3컬럼 패널 (변경 없음) |

#### 3.2.2 모바일 탭 UI

```
┌─────────────────────────────────────┐
│  ← 회의 제목              [내보내기]  │
├──────────┬──────────┬───────────────┤
│  📝 전사  │  🤖 요약  │  📒 메모      │
├──────────┴──────────┴───────────────┤
│                                     │
│  (선택된 탭의 콘텐츠 - 전체 너비)      │
│                                     │
├─────────────────────────────────────┤
│  [오디오 플레이어 - 미니]              │
└─────────────────────────────────────┘
```

- 탭 전환: 스와이프 제스처 지원 (좌우 스와이프로 탭 이동)
- 오디오 플레이어: 하단에 미니 플레이어로 고정 (재생/일시정지 + 프로그레스 바)
- 오디오 플레이어 확장: 미니 플레이어 탭 시 풀 플레이어로 확장

#### 3.2.3 요구사항

| 항목 | 설명 |
|------|------|
| 탭 컴포넌트 | 전사(TranscriptPanel), AI 요약(AiSummaryPanel), 메모(MeetingEditor) 재사용 |
| 스와이프 | 좌우 스와이프로 탭 전환 (선택, 없어도 가능) |
| 상태 유지 | 탭 전환 시 스크롤 위치/입력 상태 유지 |
| 오디오 | 미니 플레이어(56px 고정) → 탭으로 풀 플레이어 |

### 3.3 실시간 녹음 페이지 모바일 레이아웃 (P0)

MeetingLivePage도 동일한 3컬럼 구조이며, 추가로 녹음 컨트롤이 있다.

#### 3.3.1 브레이크포인트별 레이아웃

| 브레이크포인트 | 레이아웃 |
|-------------|---------|
| 모바일 (< `lg`) | **탭 전환** (전사 / AI 요약 / 메모) + 상단 녹음 컨트롤 |
| 데스크톱 (≥ `lg`) | 기존 3컬럼 패널 + 상단 컨트롤 바 (변경 없음) |

#### 3.3.2 모바일 녹음 컨트롤

```
┌─────────────────────────────────────┐
│  ← 회의 제목                  🔴 REC │
│  ⏱ 01:23:45    [⏸][⏹]    [공유]    │
├──────────┬──────────┬───────────────┤
│  📝 전사  │  🤖 요약  │  📒 메모      │
├──────────┴──────────┴───────────────┤
│                                     │
│  (탭 콘텐츠)                         │
│                                     │
└─────────────────────────────────────┘
```

- 녹음 컨트롤: 상단 고정, 핵심 버튼만 (일시정지/종료/공유)
- STT 엔진 선택, 시스템 오디오 토글 등 부가 설정은 모달/시트로 분리
- 화자 패널: 모바일에서는 별도 탭 또는 바텀 시트로 접근

### 3.4 회의 목록 페이지 (P1)

이미 반응형 그리드(`grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4`)가 적용되어 있으나, 툴바/필터 영역이 모바일 미대응.

#### 3.4.1 요구사항

| 항목 | 설명 |
|------|------|
| 검색 바 | 모바일: 아이콘 탭 시 풀 너비 검색 바 확장 |
| 필터 | 모바일: 필터 버튼 → 바텀 시트에서 필터 옵션 표시 |
| 정렬 | 모바일: 드롭다운 유지 |
| 카드 | 모바일: 1컬럼, 카드 내 정보 압축 (날짜 상대 표시, 요약 미리보기 1줄) |
| FAB | 모바일: 우하단 플로팅 버튼으로 "새 회의" / "파일 업로드" |

### 3.5 대시보드 페이지 (P1)

이미 반응형 그리드 적용. 추가 작업 최소.

#### 3.5.1 요구사항

| 항목 | 설명 |
|------|------|
| 통계 카드 | 모바일: 2×2 그리드 유지 (이미 `sm:grid-cols-2`) |
| 패딩 | `p-8` → `p-4 md:p-8` |
| 차트 | 가로 스크롤 허용 |

### 3.6 설정 페이지 (P2)

모달 기반이라 모바일 대응이 비교적 용이.

#### 3.6.1 요구사항

| 항목 | 설명 |
|------|------|
| 모달 크기 | 모바일: 풀스크린 시트로 전환 |
| 탭 내비 | 모바일: 수평 스크롤 탭 |
| 폼 요소 | 최소 높이 44px |

### 3.7 터치 최적화 (P0)

#### 3.7.1 요구사항

| 항목 | 설명 |
|------|------|
| 터치 타겟 | 모든 인터랙티브 요소 최소 44×44px (Apple HIG 기준) |
| 버튼 간격 | 인접 버튼 최소 8px 간격 |
| 스크롤 | `-webkit-overflow-scrolling: touch` |
| 호버 상태 | `@media (hover: hover)`로 호버 기능 있는 디바이스에서만 적용 |
| 텍스트 선택 | 전사/요약 텍스트는 선택 가능 유지 |

### 3.8 모바일 뷰포트 대응 (P0)

#### 3.8.1 요구사항

| 항목 | 설명 |
|------|------|
| viewport meta | `viewport-fit=cover` 추가 (노치 대응) |
| Safe Area | `env(safe-area-inset-*)` CSS 변수 활용 |
| 100vh 문제 | `h-screen` → `h-dvh` (dynamic viewport height) 사용 |
| 소프트 키보드 | 키보드 열림 시 레이아웃 붕괴 방지 (`visualViewport` API 활용) |
| 스크롤 바운스 | `overscroll-behavior: none` (풀 투 리프레시 방지) |

## 4. 페이지별 레이아웃 매핑

| 페이지 | 현재 (데스크톱) | 모바일 (< lg) | 태블릿 (sm~lg) |
|--------|--------------|-------------|--------------|
| **AppLayout** | 사이드바(w-60) + 메인 | 바텀 내비 + 메인 | 오버레이 사이드바 + 메인 |
| **DashboardPage** | 4컬럼 그리드 | 1~2컬럼 | 2~4컬럼 |
| **MeetingsPage** | 4컬럼 카드 그리드 | 1컬럼 리스트 | 2~3컬럼 그리드 |
| **MeetingPage** | 3컬럼 패널(25/45/30) | 탭 전환(전사/요약/메모) | 2컬럼(전사+요약 / 메모) |
| **MeetingLivePage** | 3컬럼 패널(20/50/30) + 컨트롤 | 탭 전환 + 상단 컨트롤 | 2컬럼 + 상단 컨트롤 |
| **SearchPage** | 리스트 + 필터 | 리스트 + 필터 시트 | 변경 없음 |
| **설정 모달** | 중앙 모달(max-w-3xl) | 풀스크린 시트 | 중앙 모달 |
