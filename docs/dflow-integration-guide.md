# 또박또박 ↔ D'Flow 연동 가이드 (쉬운 버전)

> 또박또박에서 만든 회의록을 버튼 한 번으로 D'Flow(https://wbs-web.vercel.app) 회의록 화면에 올리는 기능입니다.
> 이 문서는 **처음 연결하는 관리자**와 **매일 쓰는 사용자** 두 부분으로 나뉩니다.

---

## 한눈에 보는 그림

```
또박또박 (회의 녹음·회의록 작성)
        │  "D'Flow로 전송" 버튼
        ▼
D'Flow (https://wbs-web.vercel.app) 회의록 메뉴에 자동 등록
```

연결에 필요한 것은 딱 두 가지입니다.

| 필요한 것 | 무엇인지 | 누가 준비하는지 |
|---|---|---|
| ① 비밀키(시크릿) | 두 시스템이 서로를 믿기 위한 암호. **양쪽이 똑같은 값**이어야 함 | 또박또박 관리자 ↔ D'Flow 관리자(팀장) |
| ② 계정 이메일 | 전송하는 사람의 또박또박 이메일이 **D'Flow에도 같은 이메일로 가입**되어 있어야 함 | D'Flow 관리자(팀장) |

---

## A. 처음 한 번만: 관리자 설정 (5분)

### 1단계 — 비밀키 맞추기

D'Flow 관리자(팀장)에게 이렇게 물어보세요:

> "D'Flow에 설정된 `MINUTES_API_SECRET` 값 좀 알려주세요. 또박또박 연동에 넣어야 해요."

받은 값을 또박또박에 입력합니다:

1. 또박또박 접속 → **설정** → **연동** 탭 (관리자만 보임)
2. **활성화** 스위치 켜기
3. **서버 주소**: `https://wbs-web.vercel.app` (이미 들어있으면 그대로)
4. **API 시크릿**: 팀장에게 받은 값 붙여넣기
5. **저장** 누르기

### 2단계 — 연결 확인

같은 화면에서 **"연결 테스트"** 버튼을 누릅니다.

| 결과 | 뜻 | 조치 |
|---|---|---|
| ✅ 구분(팀)·프로젝트 목록이 뜸 | 연결 성공. 끝! | — |
| ❌ "시크릿 불일치 또는 미개통/URL 오류" | 비밀키가 서로 다름 | 1단계 값 다시 확인 (⚠️ 저장 먼저 하고 테스트해야 함 — 입력만 하고 테스트하면 이전 값으로 테스트됨) |
| ❌ "서버에 연결할 수 없습니다" | 인터넷/주소 문제 | 서버 주소 오타 확인 |

### 3단계 — 사용자 계정 확인

회의록을 전송할 사람들의 **또박또박 로그인 이메일**이 D'Flow에도 가입되어 있는지 팀장에게 확인 요청하세요. 없으면 그 사람은 전송할 때 "D'Flow에 동일 이메일 계정이 필요합니다"라는 안내를 보게 됩니다.

---

## B. 매일 쓰는 법: 회의록 보내기 (10초)

1. 회의가 **완료** 상태인 회의 상세 화면으로 이동
2. **내보내기** 버튼 → 메뉴 맨 아래 **"D'Flow로 전송"** 클릭
3. 미리보기 확인:
   - **구분(팀)**: 폴더 위치로 자동 선택됨 (예: `MES/물류` 폴더의 회의 → MES). 자동 선택이 안 되면 직접 고르기
   - **제목**: `하위폴더명-원제목`으로 자동 조립됨 (예: `물류-물류공정_260716`). 마음에 안 들면 수정 가능
4. **전송** 클릭 → "전송됨 · D'Flow에서 보기" 링크가 뜨면 성공

### 전송 후 표시

- 회의 제목 옆에 **"D'Flow ✓"** 배지가 붙습니다.
- 전송 후 회의록을 **수정하면** 배지가 **"D'Flow 재전송 필요"** 로 바뀝니다 → 다시 전송하면 D'Flow의 기존 글이 **덮어써집니다** (새 글이 생기지 않음. 몇 번을 보내도 안전).

---

## C. 자주 묻는 질문

**Q. 같은 회의를 두 번 보내면 D'Flow에 두 개 생기나요?**
아니요. 회의마다 고유 식별자가 붙어서 같은 글이 **갱신**됩니다.

**Q. D'Flow에 이미 손으로 올려둔 회의록이 있는데, 또박또박 회의와 연결할 수 있나요?**
네. 전송 창의 **"연결 관리"** 를 펼치고 **"D'Flow에서 찾기"** → 목록에서 해당 글을 고르면 연결됩니다. 이후 전송은 그 글을 갱신합니다.

**Q. 다른 서버(예: 새 PC)로 또박또박을 옮기면?**
회의 내보내기/가져오기(tgz)에 연결 정보가 함께 담깁니다. 옮긴 뒤 재전송하면 D'Flow의 같은 글이 그대로 갱신됩니다.

**Q. 전송이 안 돼요.**

| 화면에 뜨는 말 | 원인 | 해결 |
|---|---|---|
| "D'Flow에 동일 이메일 계정이 없습니다" | 내 이메일이 D'Flow에 미가입 | 팀장에게 계정 생성 요청 |
| "본문이 100,000자를 넘습니다" | 회의록이 너무 김 | 회의록을 나누거나 줄이기 (전사 원문은 원래 전송 제외) |
| "D'Flow 인증 실패" | 비밀키 불일치 | 관리자에게 A-1단계 재확인 요청 |
| 구분(팀)을 고르라고 함 | 폴더 최상위 이름이 D'Flow 팀 이름과 다름 | 목록에서 직접 선택 (한 번만, 저장 안 됨) |

## D. 관리자용 참고 (기술)

### 값 예시

**또박또박 쪽** — `settings.yaml`의 `dflow:` 섹션 (설정 탭에서 저장하면 자동으로 이렇게 기록됨):

```yaml
dflow:
  enabled: true
  base_url: https://wbs-web.vercel.app        # 뒤에 /api/v1 을 붙이지 마세요 (자동으로 붙음)
  api_secret: 3f9a1c...64자리 16진수 문자열     # ⚠️ 예시 — 실제 값은 문서에 적지 말 것
```

**D'Flow 쪽** — Vercel 프로젝트 환경변수 (팀장 담당):

```
MINUTES_API_SECRET=3f9a1c...또박또박과 완전히 동일한 값
```

- 두 값이 **한 글자라도 다르면** 연결 테스트에서 "시크릿 불일치"가 뜹니다.
- 새 비밀키가 필요하면 터미널에서 생성: `openssl rand -hex 32`
- 현재 또박또박에 저장된 실제 값 확인: 서버의 `settings.yaml` 파일을 열어보거나, 설정 탭에는 앞4·뒤4자리만 마스킹 표시됩니다(예: `3f9a…d4c8`).

**전송되는 데이터 예시** (참고용 — 자동으로 만들어지므로 직접 쓸 일 없음):

```json
{
  "user_email": "jjinie73@gmail.com",
  "date": "2026-07-20",
  "team": "MES",
  "title": "물류-물류공정_260716",
  "body_markdown": "## 회의 요약\n...",
  "external_id": "ddobak:0198c2f4-6a1b-7c3d-9e5f-2a4b6c8d0e1f",
  "on_conflict": "replace"
}
```

### 구조

- 설정 저장 위치: 또박또박 서버의 `settings.yaml` → `dflow:` 섹션 (`enabled` / `base_url` / `api_secret`)
- 인증: `Authorization: Bearer <시크릿>` + 전송자 이메일을 D'Flow `auth.users`와 대조 (2중)
- 식별자: 최초 전송 시 회의마다 UUIDv7 발급(`external_id = ddobak:<uuid>`), 이후 같은 키로 upsert(replace)
- 상세 계약: `tasks/dflow-minutes-upload/artifacts/dflow-minutes-upload-api-spec.md` (D'Flow 측 사양 포함)
- 또박또박 구현 스펙: `tasks/dflow-minutes-upload/artifacts/ddobak-dflow-sender-spec.md`
