# LLM 프롬프트 압축 + 챗 답변 압축 설계

- 날짜: 2026-06-20
- 대상: `backend/app/services/llm_prompts.rb` (+ 해당 스펙)
- 브랜치: `feat/prompt-compression`
- 목적: 매 LLM 호출에 실리는 시스템 프롬프트 입력토큰 절감(목표 A) + AI 챗 답변 출력 간결화(목표 B). 의미·동작 보존이 1순위.

## 배경

`llm_prompts.rb`는 회의록 생성·AI 챗·안건 압축·인용 마커 등 15개 시스템 프롬프트 상수를 보관한다. 그중 `REFINE_NOTES_SYSTEM_PROMPT`가 3,439자로 가장 크고, mermaid 규칙 블록이 단독 1,388자(40%)를 차지한다. 이 프롬프트는 realtime 틱마다 + final마다 매번 system으로 전송돼 입력토큰을 누적 소비한다.

챗 프롬프트(`MEETING_CHAT`/`FOLDER_CHAT`)에는 `답변은 한국어로 간결하게`만 있어 답변이 장황해질 수 있다.

## 목표 A — 프롬프트 텍스트 압축 (입력토큰↓)

15개 상수 전부 대상. caveman-compress 규칙을 수동 적용한다(.rb라 스크립트 직접 실행 불가).

### 압축 규칙
- 조사·어미 컷: `~하세요`→명사형, `~합니다`→체언 종결. 예: `사용자의 피드백을 정확하게 반영하여 회의록을 수정하세요` → `사용자 피드백 정확히 반영, 회의록 수정`
- 정중체·filler·중복 불릿 제거
- 같은 패턴 보이는 예시 여럿이면 하나만(단 아래 보존 예외)

### 절대 보존 (preserve-list)
| 보존 | 이유 |
|---|---|
| `⚠️` `[필수]` `[최우선]` + 우선순위 의미 | 지시 강도 정보 |
| ` ``` ` 코드펜스 내부 전부 (mermaid 예시·JSON 스키마·표 포맷 예시) | 모델이 그대로 복사 |
| `⟦t:<ms>/s:<화자>⟧` `⟦m:..⟧` `<<<FOLLOWUPS>>>` 센티넬 | 파싱 계약 |
| 유니코드 기호목록(`² ₂ ± × → ≤` 등) | 지시 내용 자체 |
| 스펙 앵커어(`화자값` `충실하게` `분량 제한 없이`) | rspec `include` 검사 통과 |
| Ruby `#{..}` `<<~` `.freeze` HEREDOC 구분자 | 문법 |

### mermaid 예시쌍 — 보수 채택 (승인됨)
`✅/❌` 예시쌍(노드 따옴표·`<br/>`·mindmap id 규칙)은 **그대로 둔다**. 주변 산문만 압축. 사유: 이 예시들은 실제 렌더 크래시(bare 따옴표 mindmap → 빈 화면 등)를 막는 것으로, 검증이 1회 A/B뿐이라 보수적으로 간다. REFINE 예상 절감 ~30%(예시쌍 제외 산문 기준).

## 목표 B — 챗 답변 압축 (출력토큰↓·UX)

`MEETING_CHAT_SYSTEM_PROMPT` + `FOLDER_CHAT_SYSTEM_PROMPT`의 `답변은 한국어로 간결하게` 한 줄을 강화한다:

> `답변은 한국어로 간결하게. 명사형/체언 종결(~함·~예정) 사용, 서론·맺음말·군더더기 생략. 핵심부터.`

- 하드 길이 캡 없음(승인됨) — 내용 많으면 길어질 수 있음. 정보 손실 위험 최소.
- 회의록 요약 프롬프트는 출력을 건드리지 않는다(분량은 verbosity/`VERBOSITY_CHAR_LIMITS`로 별도 제어).

## 검증 (승인: 스펙 + 1회 실전 A/B)
1. 압축 전후 **글자수 diff** 측정·기록 (상수별 + 합계)
2. **풀 rspec** 통과 — 앵커 보존 확인 (`bundle exec rspec`)
3. **실 회의 1개**(mermaid·인용 들어간 큰 회의) 구/신 프롬프트로 **재생성 A/B** → mermaid 렌더·`⟦⟧` 마커·섹션 구조 육안 비교. 회귀 없으면 통과.
