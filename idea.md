## 수정 요청사항
1. STT모델을 로컬 모델을 사용할지 서버에서 정한 모델을 사용할지를 선택하는 라디오 버튼이 개인 설정에 있어야한다. 현재 전역설정으로 되어있는데 바꿔야해. 왜냐하면 일반 사용자 권한에서는 개인설정 밖에 보이지 않아. 개인 휴대폰에서 사용하기에 개인설정에 있어야해.
2. 오프라인 회의에서 온디바이스 모델 로딩에 시간이 걸리는데, 로딩 중임을 보여주는 UI(상태 표시/스피너)가 있어야 한다.
  설계: 오프라인 모델 선로딩 + 로딩 UI
  
  동작
  - 오프라인 라이브 화면 진입 → modelDir 확정되면 자동 stt_load 선행(백그라운드)
  - 로딩 중: 하단 상태바 "온디바이스 모델 준비 중…" + 회의 시작 버튼 비활성+스피너
  - 로드 완료 → 회의 시작 활성, 누르면 즉시 녹음(멱등이라 0초 대기)
  - 로드 실패 → 에러 + 버튼 재활성(재시도)
  
  변경 (오프라인 only, 3파일)
  - useLocalRecording.ts: modelLoading 상태 + 선로딩 effect. start()는 그대로(재진입 가드 유지)
  - MobileRecordControls.tsx: isStarting? prop — 있을 때만 회의시작=비활성 스피너. 서버 경로는 prop 안 줘서 무변경
  - LocalMeetingLivePage.tsx: isStarting={rec.modelLoading} + 상태메시지

  TDD: 선로딩 effect(stt_load 1회+modelLoading 토글), MobileRecordControls(isStarting→비활성)
  
3. 오프라인 회의의 UI를 기존 온라인 회의의 UI와 동일해야 한다. 같은 UI를 사용하도록 변경
  - 음성 저장도 되어야 한다. mp3로 변환해서 저장
  - 음성 재생도 되어야 한다.
  - 다만 오프라인에서는 요약, 메모는 사용할 수 없으니 비활성화 한다.
  - 오프라인 회의 수정기능(회의명 수정)

4. UI 수정요청
  - 오프라인 회의 리스트 화면에서 '기기 저장(오프라인) ' 이라는 아무런 의미가 없으니 지우고  '오프라인 회의 시작' 버튼이 전체 라인을 차지하게 변경해줘.

5. 버그수정 요청 : 오프라인 회의 화면에서 사이드바의 설정을 눌러도 아무 변경사항이 없다. 만약 3번을 적용하면 수정이 자동으로 될수도 있겠다.

6. 회의 속성에 공유/공유 안함 옵션이 있어야겠어. 기본 옵션은 공유야. 공유를 하면 다른 사람들도 다 볼수가 있어. 다만 회의 소유자만 수정을 할 수 있으면 된다. 공유 안함 옵션을 선택하면 다른 사람이 리스트도 볼수 없으면 된다. 대시보드에도 자기회의 + 타인 소유의 공유 회의를 기준으로 숫자를 산출해서 보여주면 된다.
   - 가시 범위: 지금은 **전체 사용자 공유**(공유 시 모든 로그인 사용자 열람). admin은 god-mode 유지(비공개 포함 전체 열람·수정·삭제). 기존 회의 전부 공유 상태로 마이그레이션.

7. 오프라인 회의를 내보낼 수 있으면 좋겠다. 완전 오프라인이라 (LLM)프롬프트를 못 가져오면 전사 내용 또는 mp3를 내보내면 된다. (음성 포맷은 #3과 동일하게 mp3)
