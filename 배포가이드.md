# 또박또박 배포 가이드

## 먼저 읽기: 배포 방식 선택

또박또박은 **3개의 컴포넌트**로 구성됩니다:

| 컴포넌트 | 역할 | 기술 | 기본 포트 |
|----------|------|------|-----------|
| **Frontend** | 사용자 화면 (UI) | React + Vite | 13325 (dev) |
| **Backend** | API 서버, DB, 인증 | Rails 8 + SQLite | 13323 |
| **Sidecar** | 음성인식 (STT) | Python + FastAPI | 13324 |

이 3개를 **어디에 설치하느냐**에 따라 두 가지 배포 방식이 있습니다:

```
┌─────────────────────────────────────────────────────────────────┐
│                     배포 방식 선택                                │
├─────────────────────────┬───────────────────────────────────────┤
│  A. 서버 배포            │  B. 데스크톱 앱 (Tauri)               │
│  (여러 사람이 사용)       │  (혼자 사용)                          │
│                         │                                       │
│  서버 컴퓨터 1대에        │  내 컴퓨터에 앱 하나 설치하면          │
│  Backend + Sidecar 설치  │  3개가 전부 내장됨                     │
│                         │                                       │
│  사용자들은 브라우저나     │  인터넷 없이도 동작                    │
│  앱으로 접속              │  (로컬 STT + 로컬 LLM 사용 시)       │
├─────────────────────────┼───────────────────────────────────────┤
│  → 2장으로               │  → 3장으로                            │
└─────────────────────────┴───────────────────────────────────────┘
```


## 2. 서버 배포 (A 방식)

> **대상:** 서버 관리자. STT 서버와 웹 서버 2대에 나눠서 설치합니다.
> **사용자 컴퓨터에서는 아무것도 설치하지 않습니다.** 브라우저로 접속하기만 하면 됩니다.

### 2.1 구성도

```
[STT 서버 (GPU)]                    [웹 서버]
210.1.4.53                      10.110.1.12
┌────────────────┐                 ┌────────────────────┐
│ Sidecar :13324 │◄─── HTTP ──────│ Backend :13323      │
│ (STT 음성인식)  │                │ Nginx   :443        │
└────────────────┘                 │ Frontend (정적파일)  │
                                   │ SQLite DB           │
                                   └────────────────────┘

STT 서버에는 Python만, 웹 서버에는 Ruby + Node.js만 필요합니다.
```

### 2.2 STT 서버 설치 (Linux/macOS)

> 아래 내용은 **STT 서버**에서만 실행합니다.
> Windows인 경우 [2.2-W 항목](#22-w-stt-서버-설치-windows)으로 건너뛰세요.

#### 요구사항

| 항목 | 요구사항 |
|------|---------|
| OS | Ubuntu 22.04+ / macOS |
| Python | 3.11+ |
| RAM | 4GB+ |
| GPU | 권장 (NVIDIA CUDA) — 없으면 CPU 모드 |
| Ruby/Node.js | **불필요** |

#### 1단계: 소스 코드 가져오기

```bash
git clone <repository-url> /opt/ddobakddobak
cd /opt/ddobakddobak

# 업데이트할 때
cd /opt/ddobakddobak && git pull
```

#### 2단계: Sidecar 의존성 설치

```bash
# Python 패키지 매니저 설치
pip install uv

# 의존성 설치 (플랫폼에 따라 선택)
cd /opt/ddobakddobak/sidecar

uv sync --extra cuda     # Linux + NVIDIA GPU (권장)
uv sync --extra cpu      # Linux CPU only
uv sync --extra macos    # macOS Apple Silicon
```

#### 3단계: settings.yaml 생성

```bash
cat > /opt/ddobakddobak/settings.yaml << 'EOF'
stt:
  engine: auto                     # auto 권장 (플랫폼 자동 감지)
                                   # macOS Apple Silicon → qwen3_asr_8bit (MLX)
                                   # NVIDIA GPU → qwen3_asr_transformers
                                   # 그 외 → whisper_cpp (CPU)

# 화자분리 사용 시
hf:
  token: <YOUR_HF_TOKEN>

diarization:
  enabled: false
  similarity_threshold: 0.65
  merge_threshold: 0.55

audio:
  silence_threshold: 0.01
  speech_threshold: 0.3
  silence_duration_ms: 800
  max_chunk_sec: 30
  min_chunk_sec: 1
  preroll_ms: 300
  overlap_ms: 200
  file_chunk_sec: 30
EOF
```

#### 4단계: 실행 및 확인

```bash
cd /opt/ddobakddobak/sidecar
uv run uvicorn app.main:app --host 0.0.0.0 --port 13324

# 확인
curl http://localhost:13324/health
```

#### 5단계: 방화벽

웹 서버에서 13324 포트 접근을 허용해야 합니다.

```bash
# 웹 서버(10.110.1.12)에서만 접근 허용
sudo ufw allow from 10.110.1.12 to any port 13324
```

#### 6단계: systemd 서비스 등록 (선택)

```ini
# /etc/systemd/system/ddobak-stt.service
[Unit]
Description=또박또박 STT Sidecar
After=network.target

[Service]
Type=simple
User=ddobak
WorkingDirectory=/opt/ddobakddobak/sidecar
ExecStart=/usr/local/bin/uv run uvicorn app.main:app --host 0.0.0.0 --port 13324
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now ddobak-stt
```

#### 6단계: Windows 서비스 등록 (선택)

NSSM(Non-Sucking Service Manager)을 사용하면 Windows 서비스로 등록할 수 있습니다.

```powershell
# 1. NSSM 설치 (winget 또는 https://nssm.cc/download)
winget install nssm

# 2. uv 경로 확인
(Get-Command uv).Source
# 예: C:\Users\사용자\.local\bin\uv.exe

# 3. 서비스 등록
nssm install DdobakSTT "C:\Users\사용자\.local\bin\uv.exe" `
  "run uvicorn app.main:app --host 0.0.0.0 --port 13324"
nssm set DdobakSTT AppDirectory "D:\project\ddobakddobak\sidecar"
nssm set DdobakSTT Description "또박또박 STT Sidecar"
nssm set DdobakSTT Start SERVICE_AUTO_START

# 4. 서비스 시작
nssm start DdobakSTT

# 확인
nssm status DdobakSTT
```

> 서비스 제거: `nssm remove DdobakSTT confirm`

```nginx

    listen 443 ssl;
    server_name 회의.example.com;

        root /opt/ddobakddobak/frontend/dist;
        try_files $uri $uri/ /index.html;

    # Backend API + WebSocket
    location /api/ {
        proxy_pass http://127.0.0.1:13323;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;

        proxy_pass http://127.0.0.1:13323;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
```

## 3. 데스크톱 앱 배포 (B 방식)

> **대상:** 개인 사용자 또는 앱을 빌드해서 배포하는 개발자.

### 3.0 빌드한 앱을 사용자에게 전달하는 방법

```
[개발자 컴퓨터]                           [사용자 컴퓨터]
  npm run tauri:build
       │
       ▼
  target/release/bundle/
  ├── 또박또박.dmg          ── 전달 방법 ──→  더블클릭 → 설치
  ├── 또박또박_x64-setup.exe
  └── 또박또박.AppImage

전달 방법:
  1. GitHub Releases   ← git tag 푸시하면 CI가 자동 빌드+업로드 (권장)
  2. 파일 공유          ← Google Drive, 사내 공유 폴더 등에 업로드
  3. 웹사이트 다운로드   ← 자체 다운로드 페이지에서 배포
```

**방법 1: GitHub Releases (권장 — 자동화)**

```bash
# 개발 컴퓨터에서 태그 푸시만 하면 끝
git tag v1.0.0
git push origin v1.0.0

# → GitHub Actions가 macOS/Windows/Linux 3개 플랫폼 자동 빌드
# → GitHub Releases에 Draft로 업로드됨
# → Draft를 Publish하면 사용자가 다운로드 가능
```

**방법 2: 수동 빌드 + 파일 공유**

```bash
# 개발 컴퓨터에서 빌드
cd frontend && npm run tauri:build

# 결과물을 공유 폴더에 업로드
#   macOS: frontend/src-tauri/target/release/bundle/macos/또박또박.dmg
#   Windows: frontend/src-tauri/target/release/bundle/nsis/또박또박_x64-setup.exe
#   Linux: frontend/src-tauri/target/release/bundle/appimage/또박또박.AppImage
```

### 3.1 사용자 입장: 앱 설치만 하면 됨

배포된 앱 파일을 받아서 설치합니다:

| OS | 설치 파일 | 설치 방법 |
|----|----------|----------|
| macOS | `또박또박.dmg` | 더블클릭 → Applications에 드래그 |
| Windows | `또박또박_x64-setup.exe` | 더블클릭 → 설치 마법사 |
| Linux | `또박또박.AppImage` | `chmod +x` → 실행 |

**단, 사전에 이것들이 설치되어 있어야 합니다:**

- Python 3.11+ (`uv` 포함)
- Ruby 4.0.2

> 향후 런타임 번들링으로 이 사전 요구사항을 없앨 계획입니다.

앱을 실행하면 Backend(Rails)와 Sidecar(STT)가 **자동으로 시작**됩니다. 서버 설정이 필요 없습니다.

### 3.2 개발자 입장: 앱 번들 빌드 방법

#### 번들에 포함되는 것

`tauri.conf.json`의 `bundle.resources`에 정의되어 있습니다:

```
앱 번들 내부/
├── backend/           ← Rails 앱 소스 코드
│   ├── app/
│   ├── bin/
│   ├── config/
│   ├── db/migrate/
│   ├── Gemfile, Gemfile.lock
│   └── ...
├── sidecar/           ← Python STT 소스 코드
│   ├── app/
│   ├── pyproject.toml
│   └── uv.lock
└── config.yaml        ← 공유 설정 (기본값)
```

**번들에 포함되지 않는 것** (사용자 시스템에 있어야 함):
- Ruby 런타임
- Python 런타임 + uv
- 모델 파일 (첫 실행 시 다운로드)

#### 빌드 순서 (개발 컴퓨터에서 실행)

```bash
# 1. 사전 요구사항 확인
node --version    # 18+
rustc --version   # Rust 설치 필요 (rustup)

# 2. 프론트엔드 의존성 설치
```

## 4. 설정 파일 정리

### 설정 파일 위치

```
ddobakddobak/
├── config.yaml        ← 고정 설정 (포트, 타임아웃 등). 수정할 일 거의 없음
└── settings.yaml      ← 런타임 설정 (STT, LLM, 오디오 등). 여기서 대부분 설정
                          .gitignore에 포함 — 각 환경마다 다르게 설정
```

### settings.yaml 전체 예시

```yaml
# STT 음성인식 엔진
stt:
  engine: whisper_cpp          # whisper_cpp | faster_whisper | qwen3_asr_8bit

# LLM AI 요약 (서버 기본값 — 개인별 설정은 DB에 저장됨)
llm:
  active_preset: anthropic
  presets:
    anthropic:
      provider: anthropic
      auth_token: sk-ant-xxxxxxxxxx
      model: claude-sonnet-4-6
    openai:
      provider: openai
      auth_token: sk-xxxxxxxxxx
      model: gpt-4o-mini
    ollama:                      # 로컬 LLM
      provider: openai
      base_url: http://localhost:11434/v1
      model: qwen3.5:latest

# HuggingFace (화자분리 모델 다운로드용)
hf:
  token: <YOUR_HF_TOKEN>

# 언어 선택
languages:
  selected: [ko, en]

# 요약 주기
summary:
  interval_sec: 300

# 화자분리
diarization:
  enabled: true
  similarity_threshold: 0.65
  merge_threshold: 0.55

# 오디오 파라미터
audio:
  silence_threshold: 0.01
  speech_threshold: 0.3
  silence_duration_ms: 800
  max_chunk_sec: 30
  min_chunk_sec: 1
  preroll_ms: 300
  overlap_ms: 200
  file_chunk_sec: 30
```

### 설정 우선순위

```
환경변수 > settings.yaml > config.yaml 기본값
```

예: `SIDECAR_HOST=210.1.4.53` 환경변수가 있으면 config.yaml 값을 덮어씁니다.

### 서버 설정 vs 개인 설정

| 구분 | 저장 위치 | 관리 | 항목 |
|------|----------|------|------|
| 서버 설정 (공용) | `settings.yaml` | 관리자만 수정 | STT 엔진, LLM 기본 프리셋, HF 토큰, 오디오, 화자분리 |
| 개인 설정 | DB `users` 테이블 | 각 유저가 UI에서 수정 | LLM 프로바이더/API키/모델 오버라이드, AI 요약 on/off |

개인 LLM 설정이 없으면 서버 기본값(settings.yaml의 active_preset)으로 동작합니다.

## 6. 문제 해결

### 포트 충돌

```bash
# 이미 사용 중인 포트 확인
lsof -i :13323    # Backend
lsof -i :13324    # Sidecar
```

### Sidecar 연결 실패

Backend가 Sidecar에 연결하지 못할 때:

```bash
# Sidecar가 실행 중인지 확인
curl http://localhost:13324/health

# 다른 서버에 있다면 환경변수 확인
echo $SIDECAR_HOST $SIDECAR_PORT
```

### STT 모델 다운로드 실패 (SSL 에러 / 기업 프록시)

기업 네트워크 등에서 SSL 프록시 때문에 모델 자동 다운로드가 실패할 수 있습니다:

```
ssl.SSLCertVerificationError: certificate verify failed: self-signed certificate in certificate chain
```

**해결: 모델 파일을 직접 다운로드하여 복사**

인터넷이 되는 PC에서 모델 파일을 다운로드한 뒤 STT 서버로 복사합니다.

| STT 엔진 | 모델 파일 | 크기 |
|----------|----------|------|
| `whisper_cpp` | `ggml-large-v3-turbo.bin` | ~1.6GB |

다운로드 URL:

```
https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-large-v3-turbo.bin
```

복사 위치:

```
# Windows
C:\Users\<사용자명>\AppData\Local\pywhispercpp\pywhispercpp\models\ggml-large-v3-turbo.bin

# Linux
~/.local/share/pywhispercpp/models/ggml-large-v3-turbo.bin

# macOS
~/Library/Application Support/pywhispercpp/models/ggml-large-v3-turbo.bin
```

> 폴더가 없으면 직접 만들면 됩니다. 파일을 넣으면 다운로드를 시도하지 않고 바로 로드합니다.

### 데스크톱 앱에서 "확인되지 않은 개발자"

macOS에서 서명 없는 앱을 열 때:
- 시스템 설정 → 개인정보 보호 및 보안 → "확인 없이 열기" 클릭
- 또는 `xattr -cr /Applications/또박또박.app`

## 7. 삭제 가이드

### STT 서버 삭제 (Windows)

```powershell
# 1. 서비스 중지 및 제거 (NSSM으로 등록한 경우)
nssm stop DdobakSTT
nssm remove DdobakSTT confirm

# 2. 소스 코드 삭제
Remove-Item -Recurse -Force D:\project\ddobakddobak

# 3. STT 모델 파일 삭제 (~1.6GB)
Remove-Item -Recurse -Force "$env:LOCALAPPDATA\pywhispercpp"

# 4. Python 가상환경 캐시 삭제 (uv)
Remove-Item -Recurse -Force "$env:LOCALAPPDATA\uv"

# 5. 방화벽 규칙 제거
Remove-NetFirewallRule -DisplayName "DdobakSTT"
```

### STT 서버 삭제 (Linux)

```bash
# 1. 서비스 중지 및 제거
sudo systemctl stop ddobak-stt
sudo systemctl disable ddobak-stt
sudo rm /etc/systemd/system/ddobak-stt.service
sudo systemctl daemon-reload

# 2. 소스 코드 삭제
sudo rm -rf /opt/ddobakddobak

# 3. STT 모델 파일 삭제
rm -rf ~/.local/share/pywhispercpp

# 4. 방화벽 규칙 제거
sudo ufw delete allow from 10.110.1.12 to any port 13324
```

### 웹 서버 삭제 (Linux)

```bash
# 1. 서비스 중지 및 제거
sudo systemctl stop ddobak-backend
sudo systemctl disable ddobak-backend
sudo rm /etc/systemd/system/ddobak-backend.service
sudo systemctl daemon-reload

# 2. Nginx 설정 제거
sudo rm /etc/nginx/sites-enabled/ddobak
sudo rm /etc/nginx/sites-available/ddobak
sudo nginx -s reload

# 3. 소스 코드 + DB + 오디오 삭제
sudo rm -rf /opt/ddobakddobak
```
